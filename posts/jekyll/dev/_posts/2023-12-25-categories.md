정말 수십번 고민에 고민이 들어간 듯하다.

단순히 permalink 으로 단일 카테고리 페이지에서 처리하는 방식에서
계층형 카테고리를 구성하면서, 고심을 많이 했다.

기존 프런트 메타에 배열방식의 카테고리를 작성 후, [부모, 자식] 2레벨로 구성해서 html 의 계층 구조를 표현하기 위해서
중복제거 부모값을 가지고 온 후, 루프를 통해서 자식 html 을 만들었다.

그리고, 각 카테고리 클릭시, permalink: /categories 라는 단일 페이지로 이동이 될 수 있도록 구성하고, 
자바스크립트 data 속성을 통해서 화면을 제어할려고 하였다.

하지만, 여기서 문제점이 발생한 것이다.

카테고리의 이동 방식을 a 태그의 href 로 페이지가 제 로딩이 된다.
자바스크립트의 data 속성을 이용한다고 하더라고 /categories 페이지에서는 어떤 data 속성이 선택된지 모른다. 재로딩되니깐.....
너무 무지했다. 그리고 정적 페이지였다는 생각이 깊게 들었다.
과거에 data 속성을 활용해서 화면을 구성했던 것을 다시 보았다.
아..... 단일 페이지 내에서 화면동작을 제어하고 있었던 것이다. 
아.....

그리고 다시 고민과 고심.
카테고리별 페이지를 만들수 밖에 없는 건가 라는 생각과 함께..
_data 폴더를 통한 데이터 fix 후 사용하는 것 뿐인가? 라는 생각과 함께..

카테고리별 페이지를 만든다면 만들 수는 있지만, 내가 post 작성시, post 에도 카테고리를 작성하고, 해당 카테고리 페이지를 새로 만들고,,
너무 노가다라는 생각이 들었다. 데이터도 추가해야 되고..

한참 생각 후..

지킬(jekyll) doc 문서에서 
카테고리는 md 파일에서 프론트메타로 작성하는 방법과
폴더1/폴더2/_posts/ 작성이 되면 폴더1/폴더2 가 카테고리로 잡힌다는 내용을 보았다.
그렇다면, 폴더1 에 index.html 를 만들고, permalink 를 카테고리로 정한다면, 
어차피 카테고리별 post 파일이 구성이 되며, post 작성시, 별도 프론트메타를 작성할 필요도 없고,
링크만 카테고리이름으로 해주면, index.html 로 이동되지 않나? 
라는 생각을 하게 되었다.
테스트....
된다.

아.. 그다음은..
html 계층 표현인데...
2레벨 구성, 자식카테고리에서 2개이상의 post 파일이 있으면, 화면에서 카테고리가 2개찍히네.. ㅎ
젠장..
중복제거처리를 해야 되는데..

어떻게 또 데이터 구조를 가져가야 되나..
고심......중...............
